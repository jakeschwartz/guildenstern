---
name: Onboarding vs daily-use frame
description: The product has two design surfaces with different success criteria. Every new contact / partner / decision is its own onboarding moment. Use this frame when scoping which surface a piece of work is for.
type: feedback
---

The founder named this explicitly day two, and it's load-bearing:

> "We really have two different things here. One is great onboarding (new contacts are essentially onboarding + ongoing utility). The other is lovely daily use experience. We need to engineer both. The time will disproportionately go to these edge cases that are important because they will be the user's first."

**Two surfaces, different success criteria:**

| | Onboarding | Daily use |
|---|---|---|
| Triggers | First install; every new contact; every new partner; every new decision | The agent thread, the threads list, the briefing |
| Goal | "Oh, I get it" in under 60 seconds | "I keep opening this" |
| Voice | Slightly more expansive, explanatory | Calm, terse, present-tense |
| Friction tolerance | Lower (one chance to land) | Lower (must not annoy over time) |
| Craft per occurrence | High — disproportionate, justified | Sustainable — every screen, forever |

**Onboarding includes per-relationship onboarding.** Every QR scan opens a new relationship; each one is a first impression for *that* relationship. The chip-set, the agent's welcome message, the end-of-day review — all onboarding work. They happen rarely per app session, but they shape the user's belief about the product every time.

**Surfaces that serve both need designed transitions.** Examples to watch for:

- **Chip-set** in a relationship thread: front-and-center the first time, should fade or collapse on subsequent visits.
- **Agent's morning briefing**: should feel like magic on day one, calm-default on day thirty.
- **Agent home thread**: onboarding-flavored when empty (welcome the user), daily-use when full.

These aren't edge cases — they're real design choices that need explicit handling per surface.

**How to apply:**

- When proposing or scoping work, name which surface it serves. Many discussions get clearer when this is named ("this is onboarding work" / "this is daily-use polish").
- Onboarding work justifies more craft per-occurrence than daily-use work. A 90-second polish session on the agent's welcome copy is worth it even though it fires once. A 90-second polish on a row in the threads list earns its keep across thousands of viewings, but each iteration buys less per minute.
- After a stretch of onboarding work, deliberately pivot to a daily-use pass and ask: what does Jake's 30th morning feel like, not his first?
